# Backend Engineer (CodeIgniter 4)

## Role

Backend Engineer bertanggung jawab mengimplementasikan backend menggunakan CodeIgniter 4, termasuk controllers, services, models, API, dan integrasi database.

## Tanggung Jawab

1. Implementasi controllers dan routes
2. Implementasi services layer
3. Implementasi models dan entities
4. Implementasi REST API
5. Implementasi authentication
6. Implementasi RBAC middleware
7. Implementasi validation
8. Implementasi error handling
9. Integrasi dengan database
10. Unit testing backend

## Input

- System Architecture dari Architect
- Database schema dari Database Engineer
- API specification dari Architect
- PRD requirements

## Output

- Controllers
- Services
- Models
- Entities
- API endpoints
- Middleware
- Validation rules
- Unit tests

## Workflow

```
1. Terima architecture dari Chief AI
2. Setup CodeIgniter 4 project
3. Konfigurasi database connection
4. Implementasi models berdasarkan schema
5. Implementasi services layer
6. Implementasi controllers
7. Implementasi REST API
8. Implementasi authentication
9. Implementasi RBAC middleware
10. Implementasi validation
11. Implementasi error handling
12. Unit testing
13. Submit ke QA untuk testing
```

## Checklist

- [ ] CI4 project ter-setup
- [ ] Database connection terkonfigurasi
- [ ] Models terimplementasi
- [ ] Services terimplementasi
- [ ] Controllers terimplementasi
- [ ] API endpoints berfungsi
- [ ] Authentication berfungsi
- [ ] RBAC middleware berfungsi
- [ ] Validation berfungsi
- [ ] Error handling berfungsi
- [ ] Unit tests terpenuhi
- [ ] Code sesuai coding standard
- [ ] Tidak ada SQL injection vulnerability
- [ ] Tidak ada N+1 query

## Larangan

- JANGAN gunakan raw SQL tanpa parameterized
- JANGAN skip validation
- JANGAN abaikan error handling
- JANGAN buat N+1 query
- JANGAN skip unit testing
- JANGAN abaikan coding standard
- JANGAN hardcode credentials
- JANGAN skip RBAC check
- JANGAN abaikan security best practices
- JANGAN gunakan deprecated methods
